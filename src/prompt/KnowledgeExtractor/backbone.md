
# 🧠 知识点自动提取与分类器

## 🎯 任务目标

请仔细阅读我提供的【待处理文本】，提取其中有价值的知识点。

为了方便后续自动化程序解析，**你必须将提取出的所有知识点输出为一个严格合法的 JSON 数组（JSON Array）**。去重、不要臆造，如果不确定则省略。

## 🔍 深度提取指令 (CRITICAL)

1.  **拒绝空泛，绝对保真**：只要源文本中出现了具体的代码片段、配置参数、API 名称、确切的数据指标，必须 100% 原样保留在卡片的 `content` 字段中。“Show, Don't Tell”。
    
2.  **上下文连贯性 vs. 原子化拆分**：
    
    -   **绝不能将连贯的思路过度碎片化**。如果多个步骤构成了一个连贯的工作流（SOP），必须完整保留在同一个 JSON 对象的 `content` 中。
        
    -   **仅当**解决独立问题或松散关联时，才拆分为独立的 JSON 对象。
        
3.  **逐段地毯式扫描**：确保中间段落技术细节不被遗漏。
    

## ⚙️ 核心输出要求

1.  **纯 JSON 输出**：只输出一个合法的 JSON 数组 `[ { ... }, { ... } ]`。**绝对不要**输出任何问候语、解释性文字，**绝对不要用 Markdown 代码块包裹（不要输出 ```json）**。
    
2.  **单语输出**：仅使用中文输出内容。
    
3.  **标签纯英文**：JSON 中的 `tool`, `domain`, `architecture` 数组必须是纯英文单词。
    
4.  **字符串转义 (极其重要)**：在 `content` 字段中书写多行内容时，必须正确转义换行符（`\n`）和双引号（`\"`）。
    

## 📚 结构规范与提取维度

**注意：你只能提取下文明确提供了 JSON 模板的分类。如果某个分类的模板没有在下方出现，则严格禁止提取该类型的数据。**

### 📄 必定提取：Overview (概述)

不管文本内容如何，必须提取一个全局摘要，作为 JSON 数组的第一个元素：

{

"type": "Overview",

"title": "一句话总结的核心主题",

"tool": [],

"domain": ["纯英文领域标签"],

"architecture": [],

"content": "- **一句话摘要**：简短概括这篇文章/笔记解决了什么核心问题"

}
